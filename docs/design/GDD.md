## 3. 核心战斗机制 ✅（可复刻级规格，与代码逐条对应）

**基本盘**：双方各据一侧，中间一条河、两座桥。用圣水打出手牌，兵牌在己方半场落兵、自动过桥进攻；法术牌全场指定落点。摧毁敌方王塔立即获胜。以下所有规则均为当前实现（`logic/` 层），数值来自 `config/arena.json` 与 `config/levels.json`。

### 3.1 场地与坐标

- 逻辑层使用**抽象 tile 坐标**（与屏幕像素无关，渲染层负责映射）。场地 **18 宽 × 32 高**；y=0 是敌方底线（画面上方），y=32 是我方底线。
- **河**占 y∈[15,17) 两整行；**两座桥**在 x∈[3,5) 与 x∈[13,15)（各宽 2 格），桥面视同地面。
- 格子四种类型：出界 / 地面（可走可部署）/ 水（地面单位不可走不可部署）/ 塔占位（阻挡 + 不可部署）。
- **塔的位置与占地**（我方，敌方沿河镜像）：王塔中心 (9, 29) 占 4×4 格；左右公主塔中心 (4.5, 24)、(13.5, 24) 各占 3×3 格。
- **部署区**：我方兵牌落点必须 y≥17（己方半场），敌方 y≤15。落点还必须是可走地面（非水/塔/出界）。**非法落点 = 整次出牌被拒**（不扣圣水不动卡）。法术不受落点限制。**拆掉敌塔不解锁前场部署**（部署区恒定，与 CR 不同）。

### 3.2 时间推进与每 tick 结算顺序

- 逻辑固定 **10 tick/秒**（每 tick 0.1s），与渲染帧率完全解耦；渲染层在两 tick 之间插值。单次追帧上限 100 tick（卡顿后丢弃积压，不快进风暴）。
- **每 tick 严格按此顺序执行**（顺序即确定性，重实现必须一致）：
  1. 双方圣水回涨；
  2. 注入本 tick 出牌（单机=AI 在此出牌，玩家出牌发生在 tick 间隙；联机=服务器下发的双方指令按同一顺序应用）；
  3. 战场结算：①全体单位状态计时衰减 + 攻击冷却恢复（塔同）→ ②每个单位索敌并移动（进入攻击距离即停）→ ③软分离推挤（2 趟）→ ④攻击**收集后统一结算**（先收集所有本 tick 的攻击再一起扣血，因此同 tick 互殴可以同归于尽，结算与遍历顺序无关）→ ⑤清理死者并触发亡语 → ⑥若有塔被摧毁：释放其占位、重算全场寻路场；
  4. 对局计时 +0.1s，判定胜负。
- **全程零随机数**；出牌落点量化到 0.001 tile。这两点 + 固定顺序 = 同输入必同输出（联机同步与反作弊重放的基石）。

### 3.3 圣水与卡组

- 圣水：内部 float，**1.0/秒**匀速回涨，上限 10，**起手 0**；全场恒速（**没有** CR 的残局双倍/三倍圣水）。"够不够出牌"按向下取整的整数判断。
- 卡组 **8 张**：手牌 4 格 + 等待队列 4 张，可预览"下一张"。打出某格 → 队首补进该格、打出的牌回队尾。**不洗牌**——按组卡顺序固定循环（确定性；CR 是开局随机洗）。
- 出牌校验顺序：手牌格合法 → 落点合法（仅含出兵的牌校验）→ 圣水足够 → 扣水 → 循环卡组 → 执行卡牌积木。任何一步失败整次拒绝、状态不变。

### 3.4 卡牌 = 技能积木数组（按序同步执行）

| 积木 | 参数 | 行为 |
|---|---|---|
| `spawn_unit` | unit_id, count | 出牌点生成 count 个单位，多只按固定 8 向偏移（0.7 tile）散布；生成时套用数值乘区（我方=该卡养成，敌方=关卡难度系数，只乘血量/伤害）。**生成即参战，无部署延迟**（CR 有 1 秒部署时间） |
| `direct_damage` | damage | 打**离落点最近**的敌方单位；场上无敌兵则空放 |
| `aoe_damage` | radius, damage | 落点为圆心，圆内（欧氏距离≤radius）所有存活敌方单位吃伤害 |
| `aoe_heal` | radius, damage(=治疗量) | 圆内友军回血，不超上限 |
- 伤害/治疗积木**只作用于单位、永远打不到塔**——法术不能拆塔（CR 火球点塔是核心手段，本作刻意砍掉，拆塔必须靠兵）。
- 伤害积木可附带状态（如 郭嘉·寒江计 = radius 3.0 / damage 0 + 冻结 2.5s）。
- 升阶觉醒通过 ops 改写积木后执行（count_add 加召唤数 / num_add·num_mult 调半径伤害 / unit_field 给单位挂新字段如亡语、溅射）。

### 3.5 单位属性（units.json 字段与量纲）

| 字段 | 量纲 | 示例：虎贲校尉 / 黄巾攻城力士 / 神射黄忠 |
|---|---|---|
| hp / damage | 点 | 600/75 ｜ 2000/120 ｜ 340/110 |
| attack_speed | **攻击间隔**秒/次 | 1.2 ｜ 1.5 ｜ 1.1 |
| move_speed | tile/秒 | 1.6 ｜ 1.1 ｜ 1.6 |
| attack_range | tile | 1.2 ｜ 1.2 ｜ 5.5 |
| aggro_radius | tile，敌兵进此半径会吸引它转火 | 5.0 ｜ 5.0 ｜ 5.5 |
| body_radius | tile，推挤体积 | 0.5 ｜ 0.8 ｜ 0.45 |
| target_type | 自身是 ground/air（air=飞行） | ground |
| attack_targets | 能打 ground/air/both | ground ｜ ground ｜ both |
| target_priority | nearest（默认）/ buildings（只锁塔） | nearest ｜ **buildings** ｜ nearest |
| splash_radius / on_hit_status / death_spawn_* | 溅射半径 / 命中状态 / 亡语 | 见 3.7、3.8 |

### 3.6 索敌与移动（每 tick 重新执行）

1. **索敌优先级**：`buildings` 单位无视一切敌兵、直锁"寻路距离最近的存活敌塔"（不会被拉扯分心，反制靠身体阻挡+塔火+高单体输出）。其余单位：仇恨半径内**最近的、我能打的**敌兵 →（无）→ 最近敌塔。**每 tick 重新索敌**（永远追当前最近者，没有 CR 的"锁定到死"）。
2. **分心限制**：地面单位只对"直线可达"的敌兵分心（两点连线不穿水）——隔河的敌兵不会把它引下水，它继续走桥；飞行单位不受限。
3. **对塔寻路 = BFS 流场**：每座存活塔预计算一张全场 BFS 距离场（4 邻接、只扩散过可走地面）；地面单位每步走"距离值更小的相邻格"，自然绕桥。塔被摧毁 → 占位变为可走地面、全场重算。
4. **追击敌兵 = 直线趋向**；若下一步会踩水/出界，回退为按流场绕桥走（绝不原地卡死）。
5. **飞行单位**：任何目标都直线飞（无视水和塔占位，只挡出界）。
6. 进入攻击距离即停止移动。攻击距离判定：与目标欧氏距离 ≤ 自身 attack_range（目标是塔时再加塔半径 = max(占地宽,高)/2，即从塔边缘算起）。

### 3.7 碰撞（软分离，无物理引擎）与死亡

- 每 tick 固定 2 趟：按单位列表序两两配对（i<j），若圆形体积重叠，沿连心线各推开一半重叠量；完全重叠用确定性微偏移兜底。推挤不会把地面单位推进水/塔/界外。**飞行与地面不同层，互不推挤**。
- 血量归零即死、当 tick 清场。带亡语的单位死亡时在原地按固定 5 向偏移（0.6 tile）裂出 N 个新单位（继承己方阵营；地面裂兵自动钳到最近可走地面，防落水）。

### 3.8 攻击结算

- 冷却 = attack_speed 秒；**冷却初始为 0，接敌第一下立即出手**，之后每次攻击重置冷却。
- **命中瞬时**：出手即扣血，没有弹道飞行时间（画面上的箭矢/火球是纯演出；CR 的真弹道可以被走位躲掉，本作不能）。
- **溅射**：主目标吃伤害后，以主目标为圆心 splash_radius 内的其他敌方单位吃**同额**伤害（不打塔、遵守对空限制、主目标不重复吃）。
- **状态**（单槽，不叠加）：slow=移动与攻击冷却恢复 ×(1−幅度)；stun/freeze=移动、攻击、冷却恢复全停。新状态刷新覆盖旧状态，唯一例外：硬控（stun/freeze）生效期间不会被 slow 顶掉。
- **塔反击**：每 tick 打"射程内最近敌兵"。王塔 50 伤/1.0s 间隔/射程 7；公主塔 50 伤/0.8s/射程 7.5（从塔中心算）。塔能打空军。**王塔从开局就参战**（CR 的王塔要被打到才激活，本作没有这个机制——中路不是安全区）。

### 3.9 胜负判定

- 王塔血量归零 → 该方**立即**判负（同 tick 双王塔皆毁 = 平局）。公主塔被毁只减少该方"剩余塔血"，不结束对局。
- 180 秒到点 → 比较**双方六塔剩余血量总和**，多者胜、相等平局。**没有**加时赛、没有突然死亡、也不按"摧毁塔数"比（CR 按皇冠数+加时+先掉血判负，本作是纯血量总和一刀切）。
- PvE 另有三星判定（见 §6），不影响胜负本身。

### 3.10 联机同步与防作弊（程序向一句话）

- PvP = **锁步（lockstep）**：两端各自完整运行同一套战斗模拟，服务器只按 tick 转发双方出牌指令（TickBundle），两端同序应用；周期性将战场状态（圣水+每个单位的 id/阵营/坐标/血量/冷却+塔）量化（浮点×1000 取整）后做 SHA-256 哈希三方对帐，分叉即判定作弊/失步。非法指令（卡不在手/圣水不足/落点非法）两端按同一规则确定性丢弃。
- PvE = 出牌指令流 + 周期哈希实时上报，服务器事后用无头客户端**重放整局验算**（跨平台逐 bit 一致），谎报战果会被拒。

## 4. 卡牌体系 ✅（48 张）

**稀有度金字塔**：寻常 18 / 精良 14 / 非凡 10 / 无双 6。费用分布：1 费×3、2 费×10、3 费×12、4 费×15、5 费×6、7 费×2。

**设计铁律（本作与众不同之处）**：**稀有度 ≠ 强度**。同费用下无双卡的裸数值与寻常卡完全同尺（数值放大系数硬性 1.0）——稀有度只决定三件事：
1. **机制预算**：稀有度越高，卡的机制越独特复杂（寻常=简单可读，无双=签名机制+明确弱点）；
2. **觉醒深度**：签名觉醒只给非凡/无双；
3. **养成成本**：解锁碎片 30/50/80/120，升级升阶成本逐档翻倍——"稀有 = 难养满，不是难打赢"。

**六大流派**（组卡骨架，AI 遭遇同样按流派配组）：
| 流派 | 打法一句话 | 克制关系 |
|---|---|---|
| 数量流 Swarm | 低费人海淹没对手 | 克单体防守；被溅射/范围法术克 |
| 坦克流 Beatdown | 大坦克顶前排攒一波 | 克人海/控场；被高单体输出+拉扯克 |
| 速度流 Cycle | 极低费快速循环反复偷塔 | 克坦克流；被人海/控场克 |
| 控场流 Control | 溅射+减速高效防守转反打 | 克人海/速攻；被坦克流顶穿 |
| 空中流 Air | 空军越河直击，打没对空的 | 克无对空卡组；被对空集火克 |
| 诱饵流 Bait | 骗光对面小法术再落核心 | 克法术依赖卡组；被溅射单位克 |

每张卡入库前强制通过"**至少 2 个克制答案**"检查——不存在无解卡。数值平衡用 CV 模型（√(总血×总输出) 每费定额）先公式后手感〔待平衡：probe 数值 pass 暂挂起〕。

## 5. 卡牌养成 ✅（全部服务器结算）

| 系统 | 规则 | 数值 |
|---|---|---|
| **解锁** | 攒该卡碎片解锁入卡池 | 寻常 30 / 精良 50 / 非凡 80 / 无双 120 片 |
| **升级** | 花金币升级，加血加攻 | 每级 +10% 血/攻；成本 = 基数(80/160/320/600)×1.5^级 |
| **升阶（觉醒）** | 碎片+金币升阶（1→2→3 阶），解锁等级上限 + 觉醒能力 | 每阶数值 ×1.25；等级上限 4/7/10 级 |
| **满态** | 3 阶 10 级 | ≈ **×2.97** 基础数值（所有稀有度满态倍率相同） |
| **战力** | 队伍 8 卡战力求和，用于关卡推荐（不参与战斗结算） | 新号满编 ≈ 920 |

**觉醒设计三原则**：①改交互不改数字（觉醒后打法/克制关系变化，例：黄盖火龙鸢觉醒获得溅射，从"单体飞行肉"变"空中清场"）；②保留原弱点（觉醒不消除这张卡的克星）；③高费卡觉醒差距要小。反例禁止：纯数值膨胀、抹掉弱点的全能觉醒。
16 张非凡/无双的签名觉醒已上线 ✅（如江东镇岳巨械亡语裂兵、神射黄忠+1 弓手）；另有 5 项增强觉醒 + 连锁/灼烧/狂暴等新积木 📋 延后件。

## 6. PvE 闯关（主线）✅

- **100 关 / 10 章**，线性解锁；每章第 10 关为 **Boss 关**（难度尖峰 ×1.1 + 精英阵容 + 更凶 AI）。
- **难度曲线**：难度系数 1.0（第 1 章）→ 2.84（第 10 章），敌方兵与**敌塔血量**都随系数放大；推荐战力 800 → 2275。设计意图：前 3 章轻松练手，**第 4 章开始"咬人"**逼玩家回头养成〔待真人验证手感〕。
- **三星判定**：★获胜 ｜ ★★王塔血 ≥50% ｜ ★★★120 秒内速胜。星数记录取历史最高。
- **敌方阵容**：15 个遭遇模板（按六流派 + 亡灵海/双坦/费用倾泻等原型），逐章排布；AI 分 5 档难度（rookie→extreme）。📋 A4 将按"章节×阵营"扩容模板并回填新卡奖励。
- **进入条件**：开战前向服务器报到（在线校验卡组合法、防秒推），断线进不了战斗。

## 7. 经济系统 ✅（全部服务器权威）

| 货币 | 来源 | 用途 |
|---|---|---|
| **金币** | 首通奖励（300 起，逐章递增）、重复通关（首通 10%）、挂机 | 升级、升阶 |
| **碎片**（按卡） | 首通固定 + 重复概率掉落（每章有主打卡，如第 1 章出黄巾阴兵） | 解锁新卡、升阶 |
| **宝石** | 首通奖励（5~25） | 高级货币，用途随商店规划展开 📋 |

**挂机产出**：离线也赚金币——**50 金币/小时 × 已通章节数**，封顶 8 小时，回游戏领取。按**服务器时钟**结算，改手机时间无效。

## 8. 天梯 PvP ✅

- **流程**：主菜单「天梯征途」→ 先选卡组（存槽位）→ 匹配 → 实时对战。
- **匹配**：隐藏分 ELO（1200 起）+ 可见**杯数**（主菜单名片显示），窗口随等待放宽。
- **同步技术**：帧锁定（lockstep）+ 双端哈希对账——两台手机各自跑同一套确定性战斗，服务器仲裁，作弊改内存会立刻分叉被抓。断线可重连续打。
- **养成生效**：PvE 练的等级/阶/觉醒技能**在 PvP 完全生效**（服务器权威下发双方养成，塔不随养成缩放）🚧 待两机验收。匹配暂不考虑战力差 📋 观察后再定。
- 📋 后续：赛季重置 + 排行榜（暂缓）、战绩回放、断线 AI 托管。

## 9. 其他模式

- **新手引导** ✅：创号（起名 + 从怪物头像池选头像）→ 强制打一局引导战（压暗高亮+手指提示逐步教学）→ 进主菜单。教学战役另有 6 关保留。
- **探险（肉鸽 Roguelite）** 💤：V3 时期的单机模式——连战推进、战间三选一拿卡、遗物（圣水加速/上限+2 等）、精英/Boss 节点塔血加成、一败到底。**转型强在线后与"服务器权威"冲突，入口暂时关闭**（标注"离线原型·未开放"）；未来是否在线化改造回归，待定 📋。

## 10. 界面与体验 ✅

- **主菜单六入口**：天梯征途 / 闯关 / 养成 / 卡组 / 探险（关闭）/ 设置。基地页汇总钱包、挂机领取、队伍战力与推荐着色（绿=够打，红=该养成了）。
- **像素 UI 设计系统**：9-slice 面板按钮全屏统一；弹窗/提示走统一层级骨架（弹窗必挡下层点击，提示永不挡手）。四个长列表界面（卡组/创号/图鉴/闯关）支持按住拖动滚动。
- **养成界面**：全卡池图鉴（锁定卡显示碎片进度）、多维排序（稀有度/费用/等级/可操作）、卡详情一键升级/升阶。
- **横版战斗** 🚧：设置内实验开关（仅 PvE）——战场横放、我左敌右，为侧视帧动画素材铺路；横版 HUD 重排、切横屏、联机横版 📋 未开工。
- **开发者 GM 工具** ✅：设置内作弊面板（刷金币/解锁全卡/满养成/通关跳章），走服务器真改数据，方便测试。⚠️ 正式上线的生产环境必须移除（上线工程 E2 落实）。

## 11. 在线与公平性（策划视角一句话版）

- **服务器说了算**：钱包、卡、进度、配置全在服务器，客户端只是显示器——改本地存档/改时钟/改内存全部无效。
- **PvE 防作弊**：开战报到 + 战斗指令流实时上报 + 服务器抽查"重放"整局验算——谎报通关/秒推/未解锁卡上场都会被拒；作弊者影子标记（先记录不处罚，处罚策略后定 📋）。
- **断线体验**：掉线时战斗冻结、显示"恢复中"，重连继续；连不上则只读不可玩（明确取舍：公平与账号安全 > 离线可玩）。
