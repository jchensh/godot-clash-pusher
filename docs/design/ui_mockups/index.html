待评审08 · 引导 / 设置 / 通用弹层

三机：新手引导挖洞层 / 设置弹窗（零专属资源）/ 确认框+Toast+断线规范。

全局规范速查（详见飞书《UI 系统策划案》）

画布：逻辑宽 720 恒定；标准设计画布 720×1560；背景出血 720×1600；所有交互内容须在 720×1280（16:9 兜底）内放得下。交付切图 @2x。

安全区：顶部 100px 内禁放交互与关键信息（灵动岛/刘海带）；底部 60px 手势条带内禁放交互；装饰与背景可自由穿越。每张示意图内红虚线即此两线，可按钮开关。

交互硬规矩：热区 ≥88×88px；主操作放右手拇指区；按钮必有按下态（下沉+压暗）；列表一律滚动；弹窗压暗 55%~65%；Toast 不挡操作 2 秒自消；演出必可跳过。

导航架构：五页签（商店灰·卡牌·对战·闯关·探险灰）+ 顶部齿轮设置；旧「六按钮主菜单」「基地页」废弃——挂机金库并入主界面活动轨，闯关进度并入章节主视觉。

预览方式：直接双击打开任意 html，或跑 .claude/launch.json 里的 ui-mockups 静态服务（端口 8766）。新增示意图请复用 ui_mock.css 模板并在本页登记。
