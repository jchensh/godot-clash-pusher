# ADR-0001：近期采用单活 Gateway

- 状态：Accepted（E0，2026-07-11）
- 决策者：用户按 E0 推荐方案确认

## 背景

当前房间、连接、重连索引和 tick 历史主要位于 Gateway 进程内。直接增加副本会导致匹配后路由、重连命中和滚动发布行为不确定。

## 决策

Staging/Prod 近期只运行一个 active Gateway。API 与 verifier 可独立扩缩；Gateway 通过 readiness 和 drain 串行替换。负载证据未证明单实例不足前，不引入分布式房间状态。

## 后果

- 优点：保留当前内存状态模型，降低上线复杂度，故障语义清晰。
- 代价：Gateway 是已知单点；需进程自愈、短 RTO、容量预留和排空演练。
- 禁止：在没有状态外置/路由方案与故障测试时，把副本数调到 2+。

## 重新评估触发器

单实例达到持续 60% CPU/内存/连接容量，或业务 RTO 无法接受单点重启时，按 `GATEWAY_STATE_MODEL.md` 的多活门禁立项。
