## 用例 1 — 地形渲染（地面 / 河 / 桥）

- **入口**：两机各点主菜单「天梯对战」→ 自动登录 → 匹配中 → 配上 → 进战场。
- **操作**：进战场后静止观察场地背景（先别出兵）。
- **预期**：① 地面是**像素土黄色 tile**（不再是纯色块）；② 中线一条**蓝色动画河**（水在动）；③ 河上有**鹅卵石桥**（左右各一座可过河）；④ 河把场地分成上下两半。
- **判定**：✅ 地面/河/桥三种 tile 正确渲染、河水动画流动。
- **记录**：tile 是否对齐、有无错位/闪烁。

## 用例 2 — 单位精灵（贴图 / 朝向 / 状态 / 队伍色）★重点

- **入口**：进战场后双方都出兵（拖兵牌到己方半场）。
- **操作**：出几种不同兵（骑士/弓箭手/食人魔/哥布林/怨灵等），观察精灵贴图。
- **预期**：① 单位是**精灵贴图**（不再是彩色圆圈）；② **本方兵朝上走、敌方兵朝下走**（朝向正确，不反）；③ 兵走路时是 walk 动画、接敌攻击时切 attack 动画；④ 本方/敌方有**队伍色区分**（蓝/红）；⑤ 空军（怨灵/余烬火颅）**上浮 + 地面有阴影圈**。
- **判定**：✅ 精灵贴图正确、朝向不反、状态动画切换、队伍色可辨、空军上浮。
- **记录（关键）**：朝向有无反向？哪些兵贴图异常？

## 用例 3 — 塔渲染（贴图 / 王冠 / 血条 / 废墟）

- **入口**：战场静止时观察塔。
- **操作**：① 看初始 6 塔（双方各 1 王 + 2 公主）；② 推掉一座公主塔；③ 推掉王塔。
- **预期**：① 塔是**城堡贴图**（不再是方块）：王塔=大城堡、公主塔=小堡；② **王塔顶有金色王冠标记**；③ 塔上有**血条**（绿/黄/红随血量变色）；④ 塔被摧毁变**废墟堆**（压低 + 染暗）；⑤ 塔受击时**闪白**。
- **判定**：✅ 塔贴图/王冠/血条/废墟/受击闪白正确。
- **记录**：塔贴图长宽比是否正常（不压扁）、王冠位置对不对。

## 用例 4 — 圣水条 + 卡面 + 下一张

- **入口**：战场底部 HUD。
- **操作**：观察圣水条 + 4 张手牌 + 出不起的牌（圣水不够时）。
- **预期**：① 圣水是**分段紫条**（10 格，随时间逐格亮，满槽时脉动发光）；② 右侧有**「下一张」预览**（卡名 + 费用珠）；③ 手牌卡面**自绘**：底板 + **兵牌=单位精灵正面图 / 法术牌=特效图标** + 卡名 + 费用珠（左下紫圈数字）；④ 圣水不够的牌有**暗罩扫光**（随圣水涨落）；⑤ 选中/拖拽时卡牌**抬起 + 金边高亮**。
- **判定**：✅ 圣水分段脉动、卡面有精灵图、下一张预览、不可用扫光、选中抬起。
- **记录**：卡面图清晰否、费用珠位置。

## 用例 5 — 出兵涟漪 + 远程投射物 + 命中 FX

- **入口**：出兵 + 远程兵开火 + 法术命中。
- **操作**：① 出一个兵看落地；② 出弓箭手/女巫看射箭/法术弹；③ 出火球/闪电/箭雨命中敌兵群。
- **预期**：① 兵落地有**尘土涟漪**；② 弓箭手射出**箭矢**（飞向目标）、女巫射**法术弹**、火颅射**火球**；③ 火球命中**爆炸序列帧**、闪电**电能环扩散**、箭雨**数支箭斜插**、电火花**红电环**；④ 塔也会**射箭反击**进犯的敌兵。
- **判定**：✅ 落地涟漪、远程投射物、各法术命中 FX 区分正确、塔射箭。
- **记录**：投射物轨迹对不对、FX 有无穿模/错帧。

## 用例 6 — 战斗 juice（伤害数字 / 震屏 / 顿帧 / 火花）

- **入口**：双方交战。
- **操作**：观察单位受击、大伤害、塔被摧毁。
- **预期**：① 受击时单位上方**浮动伤害数字**（上浮淡出）；② 大伤害触发**震屏 + 顿帧**（画面短暂定格）；③ 命中有**黄色火花**（径向短线）；④ 塔被摧毁**大震屏 + 爆炸 + 顿帧**。
- **判定**：✅ 伤害数字/震屏/顿帧/火花/塔毁爆炸都有。
- **记录**：顿帧是否影响手感（联机顿帧是纯视觉、不影响 lockstep，应只冻画面不冻 sim——确认战斗不因此卡顿/不同步）。

## 用例 7 — 胜负演出 + 结果语义

- **入口**：打完一局（某方推掉王塔，或时间到）。
- **操作**：观察结算演出。
- **预期**：① 画面**调暗**；② **标题 sting**（胜利！/失败/平局）字号回弹放大淡入；③ **王冠落入**（你拆掉的敌塔数，逐个延迟下落）；④ **比分滚动**（塔血 我方:敌方）；⑤ 稍后**按钮淡入**（再战 / 菜单）；⑥ **胜负判定正确**：你赢了显示「胜利」、输了显示「失败」（不反）。
- **判定**：✅ 演出完整、胜负语义正确（不因 side 翻转而反）。
- **记录（关键）**：side1 和 side2 各验一次——胜负显示是否都正确？

## 用例 8 — side2 视角全面检查 ★★最关键

> 联机 side2 玩家整场 180° 翻转，本方半场恒在屏幕下方。这是联机特有、单机没有的视角，是本次搬运的最大风险点。

- **入口**：**换一局**，让其中一台机器这次被分到 side2（多匹配几次，或两机轮流当 side2）。
- **操作**：side2 那台机器上，把上面用例 1~7 全过一遍。
- **预期（side2 专属）**：① 本方半场（屏幕下方）**高亮 + 可部署**，拖兵落点合法绿/非法红正确；② 本方兵**朝上走**（向屏幕上方推进），敌方兵朝下；③ 本方塔在屏幕下方、敌方塔在上方；④ 圣水/手牌读的是**本方**的（不是敌方的）；⑤ 胜负显示正确。
- **判定**：✅ side2 视角下所有元素正确翻转、本方/敌方不混淆、精灵朝向不反。
- **记录（关键）**：side2 下精灵朝向是否反了？拖兵落点合法性对不对？圣水/手牌是不是本方的？
