# DESIGN_KINGDOM.md — 王国领地系统设计（城建经营 × 养成新维度）

> 2026-07-19 立项设计稿，同日用户拍板通过（四原则：塔维度/推荐资源方案/卖时间不卖上限·节奏运营策划控/服务器权威永久）。K0~K4 已实施（见 HISTORY 2026-07-19 条；K2 同日按用户反馈改 SLG 场景化主城），K5 PVP 下发 / K6 IAP 待排。背景：主界面 CR 改版已留「王国」页签口子（KAN-111）；
> 目标 = 拓展养成维度、成为 F2P 商业化（IAP）的长线收入来源。
> 定位对标：CR 的 King Level + COC/口袋奇兵式轻城建，压缩进「单页签自闭环」的体量。
> 所有数值均〔示意·待平衡〕；服务器权威（决策 48）与 lockstep 确定性（V4 红线）为硬约束。

## 1. 一句话定位

**你是汉末一方诸侯**：经营自己的城池（建造/升级/收取），城建产出反哺卡牌养成（金币），
**城防建设直接强化战场上你的三座塔**——这是王国系统输出到对战的**唯一数值维度**。

## 2. 为什么选「塔」作为对战数值维度（推荐案）

| 候选维度 | 评估 |
|---|---|
| **塔属性（推荐）** | ✅ 唯一没被现有养成占用的板块（卡有 level/rank，塔是白板）；主题严丝合缝（修城→城塔更硬）；CR 已验证（King Level）；PVE 攻防两用、PVP 可对称下发 |
| 全军百分比 buff | ❌ 与卡牌 level 乘区高度重叠，玩家感知模糊，数值排查困难 |
| 圣水/抽卡节奏 | ❌ 动战斗节奏 = 动 lockstep 确定性与平衡根基，风险最大 |
| 新资源型消耗品（战前 buff） | ❌ 每局前置操作，违背「即点即玩」，且消耗品经济容易通胀 |

注意与既有拍板的关系：KAN-76 的「塔不随**卡牌**养成缩放」不变——塔属性归**王国**这条新线管，
两条养成线各管一块，互不越界。

## 3. 核心循环

```
建造/升级建筑（花 粮草+木石 + 等待时间）
   → 产出资源（粮草/木石/金币）+ 城防提升
      → 金币反哺卡牌养成（升级/升阶）
      → 城防强化三塔 → 打 PVE/PVP 更顺
         → 通关解锁新地块与王城上限 → 回到建造
```

时间（timer）是核心节拍器：等待 → 想加速 → 宝石/IAP。这是本系统的商业化底座。

## 4. 资源体系（自成一套，与主经济区隔）

| 资源 | 来源 | 用途 | 与主经济关系 |
|---|---|---|---|
| **粮草** | 农田（按小时产出，有仓储上限） | 民生类建筑升级 | 王国内闭环，不可交易 |
| **木石** | 工坊（伐木+采石合一，简化） | 城防类建筑升级 | 王国内闭环，不可交易 |
| **金币** | 铸币坊（整合现有挂机金库） | 卡牌养成（既有用途） | 王国 → 主经济 单向输出 |
| **宝石** | 既有硬通货 | 加速计时/补资源缺口 | 主经济 → 王国 单向输入（IAP 兑换点） |

**关键闸门：金币不能买粮草/木石。** 城防（=对战数值）只能靠时间或宝石推进——付费路径清晰，
也避免金币通胀直通对战数值。

## 5. 建筑清单（P0 首版 7 个）

| 建筑 | 功能 | 数值挂点〔示意〕 |
|---|---|---|
| **王城** | 等级门：其余建筑等级 ≤ 王城等级；王城升级额外要求章节进度（第N章通关解锁 LvN 段） | Lv1~10，绑 国王征途 10 章 |
| **农田** | 产粮草/小时，登录收取（挂机式） | 产率随等级；仓储上限约束 |
| **工坊** | 产木石/小时 | 同上 |
| **粮仓** | 提高粮草/木石存储上限 | 节奏调控工具（防囤爆） |
| **城墙** | ★对战维度：**三塔 HP 加成** | +3%/级〔示意〕，Lv 上限=王城 Lv×2 |
| **箭楼** | ★对战维度：**三塔攻击加成** | +2%/级〔示意〕，同上 |
| **铸币坊** | 产金币/小时——**接管现有挂机金库**（rate = 现 `gold_per_hour_per_chapter` 曲线 × 铸币坊系数），主界面挂机入口保留、数值来源切到这里 | 平滑迁移，无体验断层 |
| 工匠营（P1） | 工匠队数量：同时施工数。默认 1 队，第 2 队 = 大额宝石/IAP | 经典 COC 付费点 |

## 6. 对战数值接口（工程口径）

- **城防汇总**：`tower_hp_mult = 1 + 0.03×城墙Lv`、`tower_dmg_mult = 1 + 0.02×箭楼Lv`〔示意〕。
  顶配约 塔 HP +60% / 塔攻 +40%〔示意，S8/KAN-87 平衡复盘时定终值〕；**天花板绑王城=绑章节**，
  数值上限可控、不失控 P2W。
- **服务器唯一权威**：等级/资源/计时全在 PG；mult 由**服务端算好定值下发**（整数分位存储，
  避免浮点 pow；两端不各自计算）——沿用 KAN-76「level/rank 权威下发 + 双端对称注入」的成熟管线。
- **PVE**：`pve_start` 响应带我方 tower mult，battle 注入我方三塔；敌塔照旧走关卡 coef。
  重放验证器同样吃下发值，反作弊闭环不变。
- **PVP**：`JoinRoomResp` 扩展 `side1/side2_tower_mult`（学 CardProgress 字段 9/10 的加法），
  双端对称注入 + state hash 对帐；匹配沿用 2026-07-02 拍板口径（完全生效、MMR 收敛、
  建房日志埋 `def[hp/dmg]` 观察口）。
- **确定性红线**：mult 进 lockstep 的路径全用「服务器定值 + 乘法」，禁 pow/本地推导。

## 7. 商业化设计（IAP 挂点，随全局 IAP 线后置实装）

| 付费点 | 类型 | 说明 |
|---|---|---|
| 加速（计时跳过） | 消耗型 | 城建 timer 的标准变现；按剩余时长定价 |
| 资源礼包（粮草/木石） | 消耗型 | 缺口补齐；与加速互补 |
| 第二工匠队 | 永久解锁 | 高价锚点单品（COC 验证） |
| 王国月卡 | 订阅型 | 产出 +50% + 每日宝石；长线留存收入主力 |
| 城池/塔皮肤 | 装饰型 | **塔皮肤在对战中可见**（双方都看得到）= 无数值炫耀型消费；与三国阵营皮肤天然契合（A3 已有塔分阵营皮肤储备） |

立场：**卖时间不卖天花板**——付费加速进度，但数值上限（王城=章节门）全服同一条线。
装饰与订阅承接长期收入，减少纯数值付费的公平性反噬。

## 8. 与宫廷/外交页签的边界（留口子不实做）

- **宫廷**（未来）：武将/官员系统——官员可「驻守」王国建筑加产出（王国界面预留灰态驻守位）；
  武将与卡牌联动另行设计。
- **外交**（未来）：联盟/社交——资源互赠、联盟援建加速（王国界面预留灰态「援建」按钮）。
- 王国 P0 完全自闭环，不依赖上述两者。

## 9. 分期施工（每期一步一确认）

| 期 | 内容 | 交付物 |
|---|---|---|
| **K0** | 数值表 + 配置 schema（`config/kingdom.json`：建筑等级/成本/时长/产出/城防曲线）+ 本设计评审定稿 | 配置 + 评审结论 |
| **K1** | 服务端：PG 表（buildings/resources/timers）+ API（get/build/upgrade/collect/speedup）+ 服务器时钟结算 + Go 单测 | 纯经营闭环（无对战接线） |
| **K2** | 客户端：王国页签点亮——城池视图（地块+建筑占位美术）、建造/升级/收取/计时 UI，EconomyStateCache 式缓存 + Events 订阅 | 可玩的经营页 |
| **K3** | 铸币坊接管挂机金库（产率来源切换，入口体验不变） | 平滑迁移 |
| **K4** | 对战接线 PVE：pve_start 下发塔 mult + 注入 + 重放验证器同步 | PVE 生效 |
| **K5** | 对战接线 PVP：JoinRoomResp 扩展 + 对称注入 + hash 对帐 + 两机验收 | PVP 生效 |
| **K6** | IAP 挂点（加速/资源包/工匠队/月卡/皮肤）——**随全局 IAP/支付线排期，不阻塞 K1~K5** | 变现闭环 |

## 10. 风险与对策

| 风险 | 对策 |
|---|---|
| 塔变硬 → 对局拖长、平衡漂移 | 终值随 S8/KAN-87 平衡复盘定；上限保守起步（可只开 HP 线，攻击线后放） |
| PVP 感知不公平 | MMR 吸收 + 建房日志观察口先行；若数据恶化，预案=天梯段位内城防分段匹配 |
| 新系统体量大、脱离主线 | P0 砍到 7 建筑单页签；美术全占位（同主界面策略）；K1 起每期独立可验收 |
| 与挂机金库重复建设 | K3 直接整合而非并存，单一产出源 |
