# BALANCE_V5_S8 — V5-S8d 平衡 pass：方法论 + 探针发现 + 留给 S8e 的口径

> S8d 用 headless AI-vs-AI probe 量化难度曲线。**核心结论：AI-vs-AI 不能作为本游戏难度的绝对裁判**
> （规则 AI 不会主动推塔 → 系统性低估玩家），probe 只用于**机制验证 + 曲线形状 + 粗异常检测**；
> 难度手感的绝对校准 = **S8e 真人验收**。本文件是平衡数据真相源，配合 [PLAN_V5.md](../PLAN_V5.md) §4/§11.3。

## 1. 当前曲线（生成自 `config/stages_spec.json`，工具 `tools/build_stages.py`）

## 2. Probe 方法（`tools/balance_probe.gd` + `tools/run_stage_balance.gd`）

逻辑层确定性无随机 → 同输入必同输出。对真 stages 每关跑 AI-vs-AI：
- 我方 = `hard` 档（最激进推塔基准）+ 养成乘区 = `recommended_power / 920`，另测 ×0.85 / ×0.70（操作弥补窗口）。
- 敌方 = 关卡 coef（单位 hp/dmg）+ coef（敌塔 HP）+ 关卡 ai_difficulty。
- 量：胜负 + 我方王塔剩血% + 时长。采样每章 首/中/boss（控时长；改 `SAMPLE_INDICES` 可跑全 100）。

## 3. 发现（2026-06-27 跑批）

1. **AI-vs-AI 不是可靠的绝对裁判**（确认两次）：
   - S8b：完全对称(同卡组/coef1.0/双 normal/1.0×) = 接近平局的**超时**（我方塔血 4950 vs 4845）——规则 AI 不主动推塔、打成僵局。
   - S8d：真关卡每关 @rec 都判"负"，但**早期关(ch1-3) 我方王塔满血 100%**——输在超时塔血总和（敌塔随 coef 略大 + 规则 AI 不推塔）。真人会主动推塔，不会这样输。
   - → probe 系统性**低估**玩家。绝对胜负不可信；手感终判 = S8e 真人。
2. **曲线形状被验证（probe 的真正价值）**：我方王塔剩血% 从 **100%(ch1~3 僵持·王塔安全)** 平滑过渡到 **0%(ch4+ 被敌方单位打爆)**，转折在 **ch4 附近（coef≈1.5）**——正是"早期可苟、中期开始必须养成"的预期梯度。
3. **rec 口径**：rec(ch1)=800 **< 新号满编 920**——早期关推荐战力低于新号已有战力 = 正确设计（前期碾压、练手）。rec 仅 UI 红绿提示、不影响战斗结算。probe 的 target=rec/920 在早期 <1.0 会把玩家打到比新号还弱，是 probe 取数的偏差、非关卡偏难。
4. **敌塔 HP 随 coef**：probe 中实际影响有限（早期 coef≈1 几乎不缩放；后期玩家先被单位打爆、轮不到推塔）。但它是 S8e 要重点验的**手感旋钮**（高 coef 时敌塔很肉，是否"推不动"难受）。

## 4. 留给 S8e 真人验收的口径（probe 答不了的）

- **rec_power 校准**：从第 1 章顺玩，红绿提示是否在"该养成了"的真实节点亮红？
- **敌塔 HP 随 coef 的强度**：高系数关敌塔是否肉到推不动？需要时一行可软化（`scale_opponent_towers` 改 `1+(coef−1)×k`）。
- **难度"咬人"点**：人类玩家在第几章开始明显需要养成？是否与 probe 的 ch4 转折吻合？
- **操作弥补窗口**：落后推荐 ~15% 高手能否过？落后 ~30% 是否确实需要养成（§4 锚点）？

## 5. S8d 决定

- 难度机制做实：**敌塔 HP 随 coef**（`Match.scale_opponent_towers`，已单测）。
- 曲线保持**生成器公式默认**（§1），**不去拟合 AI 假象**（probe 不可信绝对值）。
- probe 工具保留：机制验证 + 曲线形状 + 未来用更强"推塔型"驱动可再用。
- 数值/手感的绝对校准 → **S8e 真人**（[docs/ACCEPTANCE_V5_S8.md] 待写）。
