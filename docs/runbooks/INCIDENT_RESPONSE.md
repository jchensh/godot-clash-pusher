# 线上事件响应 Runbook

状态：E0 基线。

## 1. 分级

- SEV-1：越权/资产异常增发、密钥或 token 泄露、全服不可登录/对战、不可逆数据损坏。
- SEV-2：显著区域/功能不可用、Gateway 重连风暴、verifier/结算大面积积压、发布无法排空。
- SEV-3：局部退化、单账号异常、无用户影响的告警或容量趋势。

## 2. 首十分钟

1. 建立事件记录，指定 incident commander、沟通人和记录人。
2. 记录开始时间、影响面、最新变更、build/config/migration 版本；保全日志与指标。
3. 优先止损：暂停发布/GM/高风险写入口，必要时停止新匹配；不要删除证据。
4. 判断回滚、排空或凭据吊销；按 runbook 执行并记录每个动作及结果。
5. 对外只发布确认事实和下一更新时间，不猜根因。

## 3. 专项动作

- token/ticket 泄露：轮换签名钥、提升账号 session version、吊销 refresh family，检查日志/代理缓存并缩小泄露窗口。
- GM/经济越权：关闭路由/入口，冻结相关账号写入，按幂等账本核对；不得直接用客户端快照修库。
- Gateway 雪崩：停止新 ticket/匹配，摘 readiness，保留 live 诊断，按单活排空或冷重启；检查队列、重放、future tick 和慢消费者。
- verifier 积压：暂停新高风险奖励或降级为 pending，不在数据库事务中等待外部 Godot；扩 worker 前确认任务幂等。

## 4. 恢复与复盘

恢复前验证核心 SLI、经济不变量、无重复结算、队列回落和新旧版本一致。24 小时内形成时间线、影响、根因、促成因素、有效/无效动作与可验证整改项；整改进入 Jira 并指定 owner/期限。
