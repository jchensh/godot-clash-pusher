## V1 — 已完成
单 lane、纯几何白膜。核心对推机制（圣水回涨 / 8 张循环卡组 / 单位沿 lane 推进与碰撞 / 三塔制胜负）+ 3 种技能积木（生成 / 直伤 / AOE）+ 简单进攻型规则 AI。逻辑层与显示层彻底分离，纯逻辑层全单元测试覆盖，编辑器内可玩。
> **Step 9 安卓打包缓做**（编辑器即可体验/继续开发，需要分发到手机时再补）。

## V2 — 已完成
顺序 **A → D → B → C**：① **A** 3-lane 玩法深度 → ② **D** 美术/动画/音频换皮 → ③ **B** AI 攻防与难度分级 → ④ **C** 扩内容与数值平衡。**不做**空中/地面克制（留 V3）。细节见 [PLAN_V2.md](PLAN_V2.md)。

## V3 — 已完成
**从「技术验证物」做成「买断制轻中度单机产品」**。完成内容：战斗核心 2D 重构（河+双桥+流场绕桥+软推挤+塔反击，彻底取代 lane）+ 空军（飞兵越河 + 对空克制）+ 新积木（亡语/治疗）+ Roguelite 主轴（连战链/draft/relic/boss/meta/存档）+ 交互手感（拖拽部署/移动插值/受击反馈/震屏/胜负演出）+ 精灵美术（单位/塔/FX/投射物/地形/卡面）+ 音频骨架（资源表 + AudioManager autoload，真实素材待补）+ 难度 5 档 + 像素 UI 设计系统 + 6 屏统一 + 新手战役（CampaignState + 6 教学关 + 引导覆盖层）。**V3-9 平衡剩余子项**（数值/节奏调优 + 设置/导出/上架打磨）与 V4 早期阶段可并行做。细节见 [docs/PLAN_V3.md](docs/PLAN_V3.md) / [docs/HISTORY_V3_DETAILED.md](docs/HISTORY_V3_DETAILED.md)。

## V4 — 进行中
**联网升级 + 实时对战 + 服务端架构**。当前阶段 = **玩法验证**：账号 + 玩家档案云存档 + lockstep 实时对战 + 匹配 + 赛季 + 排行榜跑通；**前期不实现支付/IAP/正式登录/合规/云上线**。战斗权威 = lockstep + 状态哈希校验（沿用现有 `logic/` 10Hz 确定性 tick，不重写 Go 战斗逻辑）。服务端语言 = Go，协议 = WebSocket + protobuf，库 = PostgreSQL + Redis。单仓 `/server` 子目录 + `/proto` 共享 schema。商业模式长期 F2P + 内购解锁/养成（schema 预留但前期不实现）。**V3 Roguelite + 短战役保留为单人训练营**，不动。细节见 [PLAN_V4.md](PLAN_V4.md)。

## V5+ — 远期（产品化）

## 贯穿所有阶段的设计铁律（不变）
- **逻辑层 / 显示层彻底分离**；**玩家与 AI 对称**（都只向逻辑层发指令）。**V4 起：网络玩家与本地玩家也对称**（均经 `IPlayer` 抽象，对 `logic/` 透明）。
- 单位位置 V1/V2 用 lane 进度 `0.0~1.0`、V3 起改抽象 2D tile 空间，**始终不用屏幕像素**；**固定逻辑 tick（10Hz）**结算，不绑渲染帧，显示层做插值。
- 碰撞 V1/V2 用「队列前后关系」、V3 起改自写软推挤（体积半径 + 确定性分离），**始终不用物理引擎**。
- 数值 / 卡牌 **全走 JSON 配置**。
- **V4 新增**：客户端禁止权威化战斗状态——所有指令走服务端转发、状态以双方+服务端三方 hash 对帐为准（lockstep 路线）。
- **不过度设计**；一步一确认 + 每步 commit + 逻辑层步骤必配单元测试。
