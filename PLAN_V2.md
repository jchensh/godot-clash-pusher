## 1. V2 目标一句话
把 V1 的「单 lane 白膜原型」升级为「有 lane 博弈、看起来像个游戏」的版本：**3 条 lane 的策略深度 + 美术/动画/音频换皮 + 更聪明的 AI + 更多内容与数值平衡**。

---

## 2. 范围与顺序（用户 2026-06-07 确认：**A → D → B → C**）

| 优先级 | 模块 | 内容 |
|---|---|---|
| **A（核心）** | 玩法深度 | **3 条 lane** + lane↔塔拓扑（左→公主 / 中→王 / 右→公主）+ 出牌选 lane |
| **D** | 表现 / 换皮 | 美术贴图替白膜、动画、远程投射物 / AOE 特效、UI 美化、音频 |
| **B** | AI 深度 | 攻防结合、按 lane 选进攻 / 防守方向、`ai_difficulty` 难度分级 |
| **C** | 内容 / 数值 | 扩卡池、组卡（选卡组）、多关卡、数值平衡 pass |

**V2 明确不做**：
- ❌ **空中 / 地面克制**（飞兵 + `attack_targets`）——留到后续阶段（V3 候选）。
- ❌ 联机 / 匹配 / 聊天 / 防作弊（远期）。
- ❌ 安卓打包（= 原 V1 Step 9，缓做；需要分发到手机时再做，编辑器内即可体验/开发）。

> **V1 已埋好的脚手架，V2 主要是「接通」而非从零发明**：`Lane` 已 lane_index 化、`Battle` 已持 3 塔/方（2 公主在单 lane 下未启用）、`Match` 已持 lanes 数组、`ai_difficulty` 字段已存未用、`SkillSystem` 已支持多积木数组。

## 3. 施工图（按序执行，每步停下确认 + commit；逻辑步骤必配单元测试）

### A — 3-lane 玩法（核心）
| 步 | 内容 | 验收标准 |
|---|---|---|
| **V2-1** | 多 lane 逻辑层：`Battle` 持 3 条 lane + lane↔塔拓扑（lane 0 左→公主、lane 1 中→王、lane 2 右→公主）+ 公主塔被摧毁后的处理 | 单测：3 条 lane 各自独立推进/碰撞结算；中路通王塔、侧路通公主；王塔归零判负、超时比塔血规则不变；旧测试零回归 |
| **V2-2** | 多 lane 显示层 + 出牌选 lane：画 3 条 lane、6 座塔布局、点选目标 lane 的己方半场部署 | 编辑器：能在任意一条 lane 出牌、看推进、推不同的塔、分胜负（AI 先做最小适配，能在 3 lane 下出牌不报错） |

### D — 表现 / 换皮
| 步 | 内容 | 验收标准 |
|---|---|---|
| **V2-3** | 美术替白膜：单位/塔/背景换贴图（Kenney 免费包 或 AI 生成），逻辑层零改动 | 画面换皮、玩法完全不变（验证「逻辑/显示分离 → 换皮零成本」） |
| **V2-4** | 动画与特效：单位攻击/受击/死亡、远程投射物、AOE 爆点、塔摧毁 | 关键动作都有视觉反馈 |
| **V2-5** | UI 美化 + 音频 + 主菜单/结算界面（中文名/字体按需导入） | 主菜单 → 对局 → 结算 闭环 + 基本音效 |

### B — AI 深度
| 步 | 内容 | 验收标准 |
|---|---|---|
| **V2-6** | 规则 AI 升级：攻防结合（受威胁的 lane 优先防守）、按 lane 选进攻方向、`ai_difficulty` 难度分级 | 单测：不同难度行为有别；AI 既会进攻也会防守、会选 lane |

### C — 内容 / 数值
| 步 | 内容 | 验收标准 |
|---|---|---|
| **V2-7** | 扩卡池 + 组卡（玩家可选卡组）+ 多关卡 | 能选不同卡组 / 关卡开局 |
| **V2-8** | 数值平衡 pass（待 3 lane + 新内容成型后统一调） | 一局节奏合理、无明显「必胜套路」 |

## 4. 待细化（到对应步骤前再展开，先列出备查）
- **公主塔被摧毁后该 lane 的行为**：该 lane 单位转向打王塔（皇室战争式），还是仅「开路、该端无塔」？（**V2-1 前定**）
- **3 lane 的部署区 / 河道规则**：能否跨 lane 部署、落点是否仅限己方半场。（V2-1/V2-2 前定）
- **美术风格与素材来源**：Kenney 免费可商用 / AI 生成 / 自绘。（**V2-3 前定**）
- **难度分级的差异维度**：出牌频率 / 圣水阈值 / 是否防守 / 反应速度。（V2-6 前定）

> V2 锁定范围如上。超出范围（空地克制、联机、安卓打包）一律留后续阶段，见 [PLAN_GRAND.md](PLAN_GRAND.md)。
