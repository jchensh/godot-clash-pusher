> ## ⚠️ 决策 48 转向（2026-06-26）：单机 → 实时在线 F2P、服务器权威
> 用户拍板**推翻决策 47「单机本地、不依赖服务端」**，项目按**多人在线网游标准**开发：①进游戏强制登录 + 持久连接、**断线即不可玩**；②**服务器唯一权威**（账号/钱包/货币/卡养成/关卡进度/挂机/配置全在服务器 + PG DB，**服务器时钟**结算）；③配置服务器化（登录下发带版本配置包，客户端只内存 + 薄缓存、非零配置——战斗 sim 仍需 units/cards/skills/arena 在客户端算）；④客户端 = **瘦表现层**（UI + 客户端 lockstep 战斗 sim + 非权威缓存）；⑤战斗仍客户端 lockstep（服务器下发权威输入，胜负先信任客户端 + sanity 限制，复算/反作弊并入后续）；⑥复用 V4 的 Go+PG+账号+WS+lockstep。
> **本文件性质**：§4~§8、§10 的**游戏设计（战力模型/养成/经济/闯关/资源/配置表）仍有效**——只是"谁算、存哪"从客户端本地改为**服务器权威**；存储/施工以 §9 + §11 的在线版为准。本地原型 S0~S6 算法**镜像进 Go 做权威**、客户端那份保留做 UI 预览 + 战斗内计算。
>
> **开发纪律沿用 V1~V4**：一步一确认、每步 commit、逻辑步必配单测（客户端 GDScript + 服务端 Go）、不过度设计、每步同步 HISTORY.md + Jira（project KAN）。表现/真机步交真人验收。
> **执行原则**：本文件是地图不是合同，数值一律〔示意·待平衡〕。

## 1. V5 目标一句话

**实时在线 F2P 闯关养成手游（服务器权威）**：进游戏登录 + 持久连接，**100+ 关在线闯关**（配置化难度系数）+ **货币经济**（金币/碎片/宝石）+ **卡牌养成**（升级提数值、升阶解锁技能）+ **资源获取**（推关奖励 + **服务器时钟**挂机产出）。养成/经济/进度**全服务器权威**；客户端瘦表现层、战斗客户端 lockstep。F2P，IAP 支付/合规按上线节奏（V6+）。

**核心循环（F2P 留存引擎）**：

```
推关（花操作/时间）→ 掉落金币 + 卡牌碎片 → 养成卡牌（升级=数值↑ / 升阶=数值跳变 + 解锁新技能）
   → 战力↑ → 攻克更高难度系数的关 → 循环。  挂机离线产金币，平滑养成节奏 + 给回流理由。
```

四个系统不是 4 个孤立功能，而是**一条经济链**：关卡=产出口，养成=消耗口，难度系数=闸门，挂机=平滑水龙头。

## 2. 已锁定决策（用户 2026-06-26 确认）

| 维度 | 决定 |
|---|---|
| **核心范式** | **战力为底·操作为顶**（hybrid）：养成给**有意义但有上限**的数值；难度来自①数值系数②AI 行为档③敌方卡组变脏。中等战力差能靠操作弥补、巨大战力差不能 → 既保住技巧战斗的乐趣，又给养成爽感。 |
| **关卡产出** | **模板池 × 难度系数曲线**：手工 ~15 个遭遇模板（敌方卡组 + AI 行为原型）+ 配置化系数递增 + 每章主题 + boss 手工特化。 |
| **养成深度** | **10 级 × 3 阶**（浅养成）：单卡满 ≈ ×3.0 hp/dmg；养成是轻量加分层，战斗更偏操作。 |
| **难度曲线** | **线性平滑递增**（系数 1.0 → ~2.6）+ boss 小跳（×1.1）。 |
| **集卡维度** | **初始 8 张（= 现默认卡组）+ 推关逐步解锁其余 8 张**（攒碎片解锁）。 |
| **货币** | **金币（软，升级）+ 卡牌碎片（每卡，解锁+升阶）+ 宝石（高级币占位，前期只产不充）**。 |
| **升级/升阶** | **升级花金币**（平滑数值，高频）；**升阶花碎片+金币**（里程碑，数值跳变 + 解锁技能积木 + 抬等级上限）。 |
| **节奏闸门** | **挂机离线金币（按进度产出、封顶）+ 无体力**（关卡随便刷，靠重复奖励边际递减自然限速）。 |
| **平台/部署（决策 48 改）** | ~~全程单机本地~~ → **实时在线、服务器权威**：强制登录 + 持久连接、断线不可玩；账号/钱包/养成/进度/配置全在服务器 + PG；客户端瘦表现层 + 非权威缓存。复用 V4 Go+PG+账号+WS+lockstep。 |

**与 V4 的关系**：V4-S0~S4 成果（账号/档案/lockstep/匹配）保留在仓库不动；V4-S5（赛季/榜，Jira KAN-41）退回 To Do 暂缓。联机对战将来若重启，养成数据需纳入 lockstep 同步（届时再设计，不在 V5 范围）。

## 4. 战力模型（hybrid 的灵魂，全系统地基）

**机制**：出兵时单位 `hp` / `damage` 各乘一个系数；`attack_speed` / `move_speed` / `attack_range` **永不缩放**（保手感 + 确定性）。
- 我方乘区 = f(卡 level, 卡 rank)；敌方乘区 = 关卡 `difficulty_coef`。
- 敌塔 HP 随 coef 放大（或每关 `tower_hp` 显式配，二选一，S3 定）。

| 来源 | 公式〔示意·待平衡〕 | 满值 |
|---|---|---|
| **升级** Level 1→10 | 每级 hp/dmg +10%（线性叠加） | ×1.9 |
| **升阶** Rank 1→3 | 每阶 ×1.25（2 次跳变） | ×1.56 |
| **我方满养成** | 升级 × 升阶 | **≈ ×3.0** |
| **敌方难度系数** | `coef = 1.0 + (stage_global_index − 1) × 0.016` | 第 100 关 ≈ ×2.6（boss ×1.1） |

**操作可弥补窗口**（hybrid 调参核心）：
| 我方队伍战力 vs 关卡推荐 | 体感 |
|---|---|
| ≥ 推荐 | 正常局，拼操作/构筑 |
| 落后 ≤ ~15% | 兵更脆，**高手能靠走位/防守过** |
| 落后 > ~25–30% | 兵被秒，**几乎必须先养成** |

> 窗口宽窄 = (升级加成幅度) vs (系数曲线斜率) 的博弈，是 V5-S8 平衡 pass 的头号旋钮。

**worked example**：食人魔基础 2000hp。第 50 关 coef ≈ ×1.78 → 敌方食人魔 ≈ 3560hp。你练到 8 级 2 阶（×1.7×1.25 ≈ ×2.13）→ 4250hp，小优势 → 操作局；练满 ×3.0 → 6000hp → 碾压。这就是"差一级就过"的卡点钩子。

**推荐战力（power rating）**：每卡按 `base_power[rarity] × (1 + (level−1)×0.10) × rank_mult` 折算 power，队伍 = 8 卡之和；每关配 `recommended_power`；UI 显示「我的战力 X / 推荐 Y」红绿提示，告诉玩家"该养成了"。

|---|---|---|---|---|
| 骑士 knight | 普通 | ✅ | 数值跳 | 攻击附带小溅射(on-hit aoe) |
| 弓箭手 archers | 普通 | ✅ | 数值跳 | +1 弓手(2→3) |
| 哥布林 goblins | 普通 | ✅ | +1 只(3→4) | 数值跳 |
| 箭雨 arrows | 普通 | ✅ | 半径+ | 伤害+ |
| 电火花 zap | 普通 | ✅ | 伤害+ | 命中范围+ |
| 骷髅兵 skeletons | 普通 | 🔒 | +1 骨(4→5) | +1 骨(5→6) |
| 食人魔 giant | 稀有 | ✅ | 数值跳 | 亡语小溅击(死亡 aoe) |
| 怨灵 minions | 稀有 | ✅ | +1 只(3→4) | 数值跳 |
| 火球术 fireball | 稀有 | ✅ | 半径+ | 伤害+灼烧 |
| 狂战士 mini_pekka | 稀有 | 🔒 | 数值跳 | 对塔/建筑加伤 |
| 滚石 log | 稀有 | 🔒 | 滚更远(range+) | 伤害+ |
| 女巫 musketeer | 史诗 | 🔒 | 数值跳 | 射程+/穿透 |
| 余烬火颅 baby_dragon | 史诗 | 🔒 | 数值跳 | 溅射半径+ |

## 6. 货币经济账本（faucet / sink）

| 币 | 产出口（faucet） | 消耗口（sink） |
|---|---|---|
| **金币 Gold** | 首通/重复关卡、挂机、日常任务 | 卡牌**升级**（主水池） |
| **碎片 Shards**（每卡独立） | 指定关掉落、boss、星宝箱、商店 | **解锁**新卡 + 卡牌**升阶** |
| **宝石 Gems**（占位） | 首通、成就、日常、星宝箱 | 商店换碎片/金币（**前期只产不充**，IAP 留后） |

**成本曲线〔示意·待平衡〕**：

| 稀有 | 升满(1→10)累计金币 | R1→R2 | R2→R3 | 解锁攒碎片 | base_power |
|---|---|---|---|---|---|
| 普通 | ~4k | 20 片 + 2k | 50 片 + 5k | 30 | 100 |
| 稀有 | ~8k | 30 片 + 4k | 80 片 + 10k | 50 | 140 |
| 史诗 | ~16k | 40 片 + 8k | 120 片 + 20k | 80 | 190 |
| 传说 | ~30k | 60 片 + 15k | 200 片 + 40k | 120 | 260 |

设计意图：**金币是主循环**（高频升级），**碎片是稀缺门槛**（解锁 + 升阶卡点），**宝石是稀有润滑/未来变现位**。

## 7. 闯关结构（pillar ①，配置表驱动）

**章节制**：~10 章 × 10–12 关 ≈ 100–120 关，黑暗中世纪主题区：
①荒芜村庄(教学/并入新手战役) ②幽暗墓园(亡灵) ③兽人营地(快攻) ④食人魔沼泽(坦克) ⑤巫妖回廊(法术) ⑥吸血鬼城堡(均衡) ⑦龙脊山道(空军) ⑧亡者王座(综合) ⑨深渊裂隙(高系数) ⑩终末战场(boss 综合)。章末 boss 关手工特化（特殊阵容 + 抬塔血 + 高 AI 档）。

**`stages.json`（新表）每关 schema〔示意〕**：
```jsonc
"stage_2_3": {
  "chapter": 2, "index": 3,
  "encounter": "tank_push_a",        // 引用遭遇模板
  "difficulty_coef": 1.18,           // 敌方数值乘区
  "ai_difficulty": "normal",         // 复用现有 rookie→extreme 5 档
  "recommended_power": 1200,
  "stars": [{"goal":"win"}, {"goal":"king_hp_pct","min":0.5}, {"goal":"time_under","sec":90}],
  "first_clear": {"gold":500, "gems":10, "shards":{"minions":5}},
  "repeat": {"gold":60},
  "shard_drop": {"minions":{"chance":0.3,"amount":1}}
}
```

**`encounters.json`（遭遇模板池 ~15 个）**：按原型分类——坦克推进 / 快攻群海 / 空中突袭 / 法术压制 / 均衡控制 / 亡灵海 / 双坦肉盾 / 远程风筝 / 费用倾泻 / 分推骚扰……每章选模板子集 × 系数递增；boss 关手工特化。
```jsonc
"tank_push_a": { "deck": ["giant","musketeer","minions",...], "archetype": "tank" }
```

**星级**：每关 3 星（1 星=通关，2/3 星=保王塔≥50% / 限时 / 不损公主塔等）；章内星数累计 → 解锁章节宝箱（碎片 + 宝石）。

## 8. 资源获取（pillar ④）

- **推关**：首通=大额一次性（金币 + 碎片 + 可能解锁新卡 + 宝石）；重复=小额金币（**边际递减**，鼓励刷但不无限刷）。
- **挂机（离线产出）**：金币/hr ≈ f(最高通关章节)，离线累计**封顶**（如 8h），回游戏一键领。**纯本地时钟**（存 `last_collect_ts`），无需服务端。
- **日常任务 / 成就**：任务给金币+宝石，成就里程碑给宝石+碎片（V5-S6 占位，可后续充实）。
- **星级宝箱**：章内星数里程碑 → 碎片 + 宝石。
- **节奏闸门**：前期**无体力**，靠重复关边际递减 + 挂机封顶自然限速；体力/门票留作后续可选（偏变现，玩法验证期不上）。
